Strukturierte Programmierung

Niemand mag Spaghetticode

Strukturierte Programmierung

Der Anreiz:
  • Quellcode übersäht mit GOTO-Anweisungen soll vermieden werden
  • Beschränkung auf minimalistischen Funktionsumfang
  • Beispielsprache: Pascal
Kerneigenschaften:
  • sequentiell ausgeführter Code
  • meist im Zusammenspiel mit prozeduralen Eigenschaften
  • Verzweigungen zur Flußkontrolle
  • Schleifen für Mehrfachausführung/ Iteration

Strukturierte Programmierung

Ein Pseudo-Code Beispiel:
// Programm, welches eine Zahl 'base' hoch 'exp' rechnet
// Negative Exponenten werden nicht unterstützt
var base, exp Integer;
base  = Read;
exp   = Read;
if exp == 0:
  base = 1;
else:
  while exp > 1:
    base  = base * base;
    exp   = exp - 1;
Write base;

Prozedurale Programmierung

Code zum wiederverwenden

Prozedurale Programmierung

Der Anreiz:
  • Schlecht wartbare Programmmonolithen sollten lesbarer werden
  • Doppelt geschriebener Code ist doppelt fehleranfällig
  • Beispiel-Sprache: C
Kerneigenschaften:
  • Aufsplitten von Problemen in Teilprobleme
  • Aufruf von Unterprozeduren durch definierte Schnittstelle
  • Übergabe von Werten an Unter-/ Oberprozeduren
  • Unterscheidung zwischen lokalen und globalen Variablen

Prozedurale Programmierung

Nochmal das Pseudo-Code Beispiel, mit Prozeduren:
// Prozedur, welche eine Zahl 'base' hoch 'exp' rechnet
// Negative Exponenten werden nicht unterstützt
proc Pow(base, exp Integer) Integer:
  if exp == 0:
    return 1;
  while exp > 1:
    base  = Sqr(base);
    exp   = exp - 1;
  return base;

proc Sqr(base Integer) Integer:
  return base * base;

Objektorientierte Programmierung

Intuitive Weltabbildung

Objektorientierte Programmierung

Alle Probleme der Computertechnik können durch eine weitere Schicht der Abstraktion gelöst werden.

Der Anreiz:
  • Abbilden des betreffendes Teils der Realität in der Anwendung
  • Simples erweitern von bestehenden Programmkonstrukten
  • Verstecken komplexer Algorithmen
Kerneigenschaften:
  • Klassen-/ und Objektkonzept
  • Methoden anstelle von Funktionen
  • Weitergabe von Eigenschaften & Verhalten durch Vererbung
  • Hierarchische Struktur grenzt Schnittstellen ab
  • Sichtbarkeitsmodifikatoren bestimmen Zugriffsrechte

Objektorientierte Programmierung

Erneut ein Pseudo-Code Beispiel:
// Ausschnitt eines OOP Taschenrechners
// Dieser kann jedoch lediglich die Grundrechenarten
class Calculator:
  public Calculator():
    return this;

  public Add(x, y Integer) Integer:
    return x + y;

  public Substract(x, y Integer) Integer:
    return x - y;

  public Multiply(x, y Integer) Integer:
    return x * y;

  public Divide(x, y Integer) Integer:
    return x / y;

Objektorientierte Programmierung

Alternatives Pseudo-Code Beispiel:
/* Ausschnitt eines OOP Taschenrechners
   Dieser kann jedoch lediglich die Grundrechenarten
   Dafür nutzt er Standard-Bibliotheken, um zum Beispiel Division durch Null besser zu verarbeiten
*/
from "Math" include:
  "Adder"
  "Substracter"
  "Multiplier"
  "Divider"

class Calculator extends Adder, Substracter, Multiplier, Divider:
  public Calculator():
    return this;
